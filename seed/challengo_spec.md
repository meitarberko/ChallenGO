# CHALLENGO - FULL SPECIFICATION DOCUMENT
The application is designed to provide positive daily challenges.
Users receive one randomized challenge per day.
Completing a challenge rewards the user with points.
Users can upload an image as proof of completion.
Each challenge includes a predefined hashtag.
Users can view a global community feed.
Users can like and comment on posts.
The system uses Firebase Authentication.
Firestore is used as the main database.
Room database is used for local persistence.
Notifications are used to improve engagement.
The architecture follows MVVM principles.
The UI follows Material 3 guidelines.
The app supports dark mode.
Users can edit and delete their posts.
The system prevents duplicate hashtag counting.
Each challenge remains active for 24 hours.
Users can track their progress over time.
Security rules protect user data.
The system is scalable and modular.
